# 2026-06-22 — HUD rework: honest data + real Control nodes

## Why

The codex pass had translated the static HTML mockup (`arts/ref/handoff/`) 1:1 into a single
immediate-mode `QuantumHud.gd` that drew every screen and hit-tested buttons via a hand-maintained
`_buttons` rect map. Two systemic problems resulted: (1) ~2/3 of visible "controls" were drawn but
never registered, so they were silently dead (all Settings/Codex controls, Level-Select orbit nodes,
particle tray); (2) on-screen info was fabricated and ignored real state (hardcoded `QED · 02 湮灭之门`
while level 007 was loaded, fake hints/steps/stats/progress).

Decision (with user): **honest-minimal** scope + **refactor to real Control nodes**.

## What changed

- **Input precedence**: `InputRouter` now listens on `_unhandled_input` instead of `_input`, so GUI
  Controls consume clicks they own before gameplay sees them. No more per-screen hit-test bookkeeping;
  buttons can't be dead.
- **Data**: added `display_code` / `display_name` (Chinese, player-facing) to `LevelSpec` with
  fallback to `title`/`level_id`; populated all 7 `.tres`. New `LevelCatalog.gd` is the single ordered
  play list (count/spec_at/index_of/next_of/has_next).
- **HUD**: `QuantumHud.gd` is now a thin screen-stack coordinator (kept the runtime-binding +
  level_loaded/level_complete + visibility-sync logic). Screens are code-built real Controls under
  `ui/screens/`: `MenuScreen`, `LevelSelectScreen`, `PuzzleHud`, `VictoryScreen`. Shared look via
  `ui/UiTheme.gd` (Theme + type variations) and `ui/UiFactory.gd`. Decorative art kept as widgets
  (`ui/widgets/ParticleOrb`, `LogoMark`, `DiagramPreview` — the preview renders the real level graph).
- **Honesty**: Codex and full Settings screens removed (no backing system — deferred). Puzzle title +
  status read from runtime/`GraphModel.is_complete()`; fake warning/step counter/particle tray gone.
  Victory shows only real facts (level name, has-next). Menu "Continue" disabled until a level is
  played; mislabeled "恢复默认" and the non-functional "按任意键开始" removed.
- **Main**: no longer auto-loads level 007; HUD opens on the menu and loads on demand.

## Verification

- `scripts/run_gdunit.ps1`: 35/35 pass (added `test_level_catalog`, `test_quantum_hud` integration;
  existing graph/interaction/runtime suites still green). Needed a one-time `--headless --import` to
  build the gitignored `.godot` class cache before the runner could resolve gdUnit class_names.
- `--headless --quit-after 60` boots Main with no script errors.

## Deferred (I1+)

Particle palette/placement, codex/discovery, full settings (audio/graphics/a11y/language),
step/time/star scoring, save/persistence. Interfaces (`LevelCatalog`, display fields) are left in
place; systems are not implemented.

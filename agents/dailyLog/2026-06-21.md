## Iteration 0 Review and Interaction/Render Rework

Reviewed prompts 00-05. The skeleton (graph model, commands, serialization, arc-length pulse) was
solid, but the I0 *feel* was missing and hidden by green-but-shallow tests (no test ever called
`pointer_moved`). Fixes:

- **P1 (critical) — no live drag feedback.** `CurveInteraction.handle_pointer_moved` only stored a
  position; all changes applied once on `pointer_up`. Now the model updates live 1:1 during the drag
  and the whole gesture commits as **one** undo entry on release; `cancel` rewinds the preview.
- **P2 — bend inserted hard corners.** Each move inserted a new point (kinks, unbounded growth). Now a
  drag edits a single control point and auto-smooths its Bézier tangents (Catmull-Rom style).
- **P5 — anchors were draggable.** `hit_test_node` now skips `NodeKind.ANCHOR` and constrained nodes.
- **Renderer.** `CurveRenderer` now draws node handles, subscribes to `node_changed`, and makes an
  edge endpoint connected to a socket follow its node (node position is truth).
- **`ConnectHalfEdgeCommand`** aligns the snapped endpoint onto the socket and restores it on undo.
- **Tests (were absent).** Added live-drag / one-undo / smooth-bend / locked-anchor / cancel-rewind
  cases for interaction, and node-handle / endpoint-follow cases for the renderer.
